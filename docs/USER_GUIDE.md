# DetForge Studio 用户使用手册

DetForge Studio 是面向缺陷检测数据运营的 Web 工具：从平台库查询检测明细、规则/Python 筛选与采样、模型批量预测、人工质检归档，并导出 COCO / CSV。

## 1. 快速开始

### 启动服务

```bash
cd DetForge-Studio
pip install -r requirements.txt
cd frontend && npm install && npm run build && cd ..
python app.py
```

浏览器打开：**[http://localhost:5050](http://localhost:5050)**

### 开发模式（前端热更新）

```bash
# 终端 1
python app.py

# 终端 2
cd frontend && npm run dev
# 访问 http://localhost:5173（API 代理到 5050）
```

### 第一次使用建议顺序

1. **设置** → 配置数据库与图片路径 → 测试连接
2. **查询** → 选时段 → 预览筛选 → 执行查询 → 导出 COCO
3. **模型** → 从部署/训练导入权重到注册表
4. **任务** → 创建预测作业
5. **人工质检** → 拖入客户图 → 查 SN → 选中匹配 → 归档

## 3. 首次配置

进入 **设置**（`/config`）。

### 3.1 数据库


| 字段            | 说明                                |
| ------------- | --------------------------------- |
| 主机 / 用户 / 数据库 | 连接 `vision_backend` 平台库           |
| 密码            | 可留空保持原值；**推荐**用环境变量 `DB_PASSWORD` |


点击 **测试连接** 确认可用。

### 3.2 图片路径

- **拼接模式**（默认）：`img_base_path` + SQL 结果中的 `origin_object_key`  
- **完整路径模式**：直接使用 `local_pic_url` 等字段

若图片无法显示，在设置页点击 **从数据库同步路径**，或检查 `img_base_path` 目录是否存在。

### 3.3 检测项目

- **defect_approach_id**：决定缺陷类别下拉列表（默认 18）  
- 点击 **刷新类别** 从平台拉取最新标签

### 3.4 DetForge / 安全（可选）


| 字段                     | 说明                       |
| ---------------------- | ------------------------ |
| forge_database         | 写库名，默认 `detforge`        |
| device_max_concurrency | 同 GPU 设备上并发预测作业数，默认 1    |
| API Token              | 非空时所有 `/api/`* 需携带 Token |


启用 Token 后，浏览器控制台可设置：

```javascript
localStorage.setItem('pc_api_token', '你的令牌')
```

## 4. 数据查询

### 4.1 数据源

查询页顶部 **数据源** 下拉：

- **检测明细表**：平台 `product_detection_detail_result`（默认 SQL）  
- **预测结果表**：`detforge.predict_result`，用于对已预测结果二次筛选

切换为「预测结果表」时，SQL 会自动替换为预测结果表名；结果缩略图会显示 **模型预测框**（服务端渲染）。

### 4.2 基本流程

1. 选择 **策略** 或使用自由查询
2. 设置 **开始/结束时间**（支持快捷：今天、昨天、近 7 天…）
3. 编辑 **SQL**（`${START_TIME}` / `${END_TIME}` 会被替换）
4. 配置 **规则筛选** 或 **Python 代码**
5. 设置 **随机采样** 条数与种子
6. 点击 **预览筛选** 查看行数
7. **执行查询**（`Ctrl/Cmd + Enter`）

### 4.3 筛选模式


| 模式  | 说明                                       |
| --- | ---------------------------------------- |
| 规则  | 表格编辑类别、置信度、剔除比例，编译为 `apply_filter_rules` |
| 代码  | 手写 `process_data(df)`                    |
| 对照  | 规则表 + 代码编辑器并排                            |


### 4.4 结果区

- **网格 / 详情视图**：切换缩略图与单条大图  
- **筛选栏**：按 SN、型号、部位、类别、score、文件名过滤  
- **全屏 / 分页**：大量结果时分页浏览  
- **导出**：COCO ZIP、CSV、归档到服务器目录

### 4.5 策略快捷操作

- 顶栏 **保存 / 导出 / 导入** 当前工作区  
- 策略下拉：加载内置或自定义策略  
- **Pipeline 导入**：从产线 Pipeline 导入最低阈值规则

## 5. 模型管理

路径：**模型**（`/models`）

### 5.1 注册表

已登记可用于预测的模型，包含：

- 框架（`dino` / `hq_det` + 子类型）  
- 权重路径 `checkpoint_path`  
- 默认 threshold / device / max_size

操作：

- **健康检查**：尝试加载权重并立即释放（需本机有 torch/GPU 环境）  
- **启用/禁用**、**删除**

### 5.2 从平台导入


| 标签页  | 来源                 | 说明                                |
| ---- | ------------------ | --------------------------------- |
| 部署模型 | `modeldeploy`      | 路径通常可直接使用                         |
| 训练配置 | `modeltrainconfig` | `model_store_key` 为相对路径，需人工确认完整路径 |


点击 **导入** 写入注册表。

### 5.3 手动登记

在注册表上方表单填写名称、框架、权重绝对路径后提交。

## 6. 预测任务

路径：**任务**（`/jobs`）

### 6.1 创建预测作业

1. 选择 **已注册模型**
2. 选择图片来源：
  - **查询 task_id**：查询页执行后得到的 `task_id`（读 `exports/<task_id>/result.csv`）  
  - **图片目录**：服务器上某文件夹  
  - **路径列表**：每行一个绝对路径
3. 可选：阈值、设备（`cuda:0`）、max_size、任务内并发
4. 点击 **创建作业**

### 6.2 作业状态


| 状态           | 含义           |
| ------------ | ------------ |
| 排队 pending   | 等待 worker 领取 |
| 运行中 running  | 正在预测         |
| 已暂停 paused   | 可继续          |
| 完成 done      | 全部或部分成功      |
| 失败 failed    | 作业级失败        |
| 已取消 canceled | 用户取消         |


操作：**暂停 / 继续 / 取消 / 重试失败项**

### 6.3 作业详情

点击 **详情** 可查看：

- **预测结果**：带框缩略图，点击放大；支持加载更多  
- **失败项**：每张图的报错与重试次数  
- **导出结果**（人工质检导出类作业）：目录路径与 **下载 ZIP**

Worker 随 Flask 自动启动；独立运行：`python worker.py`

## 7. 人工质检

路径：**人工质检**（`/manual-qc`）

用于：客户提供拍摄图 + SN，与平台缺陷图人工核对后归档，并按成像情况分类。

### 7.1 单条核对（推荐）

1. **拖入客户图**（或点击选择）
2. 输入 **SN**，点 **查图**
3. 在 **图廊** 中点选对应的平台缺陷图；或点 **标记为拍不到**
4. 填写 **缺陷类型**、**成像情况**（必填）
5. 点 **归档**（`Ctrl/Cmd + Enter`）

**快捷键**（图廊区）：


| 键                | 作用    |
| ---------------- | ----- |
| ← / → / ↑ / ↓    | 切换候选图 |
| N                | 标记拍不到 |
| Enter            | 切换选中  |
| Ctrl/Cmd + Enter | 归档    |


**对比**：点击缩略图或右上角 ⤢ 打开客户图 ↔ 平台图并排放大。

### 7.2 批量归档

1. 拖入多张客户图
2. 在表格中逐行填写 SN、缺陷类型、成像情况
3. **批量归档**（按 SN 自动取最新一条平台图）

### 7.3 类别配置

展开 **类别配置 / 维护**：

- **成像情况**：默认 成像清晰 / 成像不清 / 拍不到  
- **缺陷类型**：可自由增删  
- **严格模式**：开启后缺陷类型只能从列表选择（前后端均校验）  
- **清理未引用客户图**：删除 `uploads/` 下未被任何归档引用的文件

### 7.4 查询与导出

- 按 **时段 + 成像类别 + 缺陷类型** 查询已归档记录  
- **导出到目录**：写入 `exports/manual_qc_export/<时间戳>/`  
- **下载 ZIP**：浏览器直接下载  
- **后台导出**：大批量时创建作业，到 **任务** 页详情下载

目录结构：`<类别>/<SN>__<id>__platform|customer.<ext>` + `manifest.csv`

### 7.5 最近归档

支持 **编辑**（成像/缺陷/备注）和 **删除**。

### 7.6 去重说明

同一 **SN + 客户图** 默认不允许重复归档；勾选 **强制归档** 可绕过。数据库层亦有唯一约束。

## 9. 导出与归档

### 9.1 查询结果导出


| 格式       | 说明                                |
| -------- | --------------------------------- |
| COCO ZIP | 含标注与图片（task 目录内副本）                |
| CSV      | 表格字段导出                            |
| 归档到目录    | 复制 COCO + 图片到 `archive_base_path` |


### 9.2 预测结果二次利用

1. 任务页等待预测 **完成**
2. 查询页数据源选 **预测结果表**
3. SQL 按时间/SN 筛选 → 规则筛选 → 导出评测集

内置策略 **误检水平评测集**（`fp_eval_set`）可按款型分层采样。

## 10. 安全与运维

### 10.1 环境变量（推荐）


| 变量                                                    | 说明                  |
| ----------------------------------------------------- | ------------------- |
| `DB_HOST` / `DB_USER` / `DB_PASSWORD` / `DB_DATABASE` | 数据库连接               |
| `DB_PASSWORD`                                         | 覆盖 config.json 中的口令 |
| `PC_NO_WORKER=1`                                      | 禁用 Flask 内嵌 worker  |
| `PC_WORKER_CONCURRENCY`                               | worker 并发数          |
| `PC_MAX_CONTENT_MB`                                   | 上传体积上限（默认 256MB）    |
| `PC_DEV_CORS=1`                                       | 开发跨域                |


### 10.2 路径安全

- 图片预览 `/api/image?path=` 仅允许 `img_base_path`、`uploads`、`exports` 内路径  
- 人工质检导出目录默认只能在 `exports/` 下

### 10.3 数据目录


| 路径                   | 内容                    |
| -------------------- | --------------------- |
| `exports/`           | 查询 task、COCO 导出、QC 导出 |
| `uploads/manual_qc/` | 拖入的客户图                |
| `config.json`        | 本地配置（勿提交 git）         |
| `detforge` 库         | 模型、作业、预测结果、质检归档       |


### 10.4 初始化写库

首次使用 **任务** 或 **人工质检** 时，若提示 schema 未就绪，页面会引导 **初始化 detforge 库**；或调用 `POST /api/forge/schema/init`。

## 11. 常见问题

### 图片显示「无法加载」

1. 检查 **设置** 中 `img_base_path` 是否存在
2. 确认 SQL 结果含 `origin_object_key` 或 `local_pic_url`
3. 尝试 **从数据库同步路径**

### 预测作业一直排队

1. 确认 worker 已启动（Flask 日志或 `python worker.py`）
2. 检查 `detforge` 库是否已初始化
3. 查看作业详情 **失败项** 错误信息

### 模型健康检查失败

- 本机需安装 torch 及对应框架依赖（DetUnify-Studio）  
- 确认 `checkpoint_path` 文件存在且可读

### 人工质检查不到 SN 的图

- SN 需在平台 `product_detection_detail_result.product_no` 中存在  
- 图多时点 **加载更多**（每次 50 条）

### 启用 API Token 后 401

在浏览器设置 Token（见 [3.4](#34-detforge--安全可选)），或请求头加 `X-API-Token`。

### 构建后页面 503

执行 `cd frontend && npm run build` 生成 `frontend/dist`。
